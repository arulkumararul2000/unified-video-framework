EmailAuth Configuration Test

Test Configuration Options

This page demonstrates the new EmailAuth configuration options:

  • allowBackdropClose: Controls whether clicking outside the modal closes it
  • showCancelButton: Controls whether the cancel button is displayed
  • placeholderColor: Controls the color of input placeholders

Test Scenarios

1. Default Configuration

emailAuth: { enabled: true, ui: { // Default behavior: backdrop close enabled, cancel button hidden // placeholderColor uses default value } }

2. Show Cancel Button

emailAuth: { enabled: true, ui: { showCancelButton: true, placeholderColor: '#00aaff' } }

3. Backdrop Close Disabled

emailAuth: { enabled: true, ui: { allowBackdropClose: false, placeholderColor: '#ffaa00' } }

4. Cancel Button with Backdrop Close

emailAuth: { enabled: true, ui: { showCancelButton: true, allowBackdropClose: true, placeholderColor: '#00ff00' } }

5. Backdrop Close Disabled (Most Secure)

emailAuth: { enabled: true, ui: { allowBackdropClose: false, // showCancelButton: false (default) placeholderColor: '#ff0000' } }

Instructions

  1. Click each test button to simulate different configurations
  2. For backdrop close tests: Try clicking outside the modal
  3. For cancel button tests: Check if the cancel button appears
  4. For placeholder color tests: Observe the input placeholder colors
